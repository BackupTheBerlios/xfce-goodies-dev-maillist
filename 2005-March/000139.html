Hei, I thought of another addition to the new API:

gboolean xfce_panel_plugin_test_availability(gchar
**return_msg);

The panel could call this function when a user
selects "Add new item" for each plugin and when
a particular plugin returns FALSE it could be
grayed out (disabled). The plugin would store an
explanation of the reason why it could not be
added this time and it might be displayed somehow
to the user.

Excuse me with all my stupid ideas, but I'm just
trying to think in terms of usability (that's a
VERY VERY modern word these days :-) ) and I think
it's quite more appropriate not to be able to add
some plugin under certain curcumstances than to be
able to add the plugin and it not to function
properly. ;-)

Again, sorry for the crap and for my not very good
english. Just needed to share these thoughts.
